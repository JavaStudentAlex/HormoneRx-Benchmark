# HormoneRx Benchmark

An open evidence dataset and benchmark for testing whether AI systems can recognize hormonal medication context and retrieve verified interaction evidence **without generating unsupported clinical advice**.

This is a research prototype built for a hackathon. The reusable scientific contribution is the **dataset + synthetic benchmark + reproducible evaluation pipeline**. The web app only demonstrates them.

> **Research prototype. Not medical advice.** Evidence is limited to the curated prototype dataset and requires verification against the cited source and individual clinical context.

## What it is

- **Evidence dataset** — six source-linked interaction records between hormonal contraceptives and hepatic enzyme inducers (plus the reversed-direction lamotrigine interaction). Every medical statement is a close paraphrase of a cited authoritative source (FSRH, CDC US-MEC, FDA lamotrigine label, MHRA).
- **Synthetic benchmark** — 20 labelled consultation snippets with immutable gold labels, covering clear/implicit positives, true negatives, explicit negation, historical use, and ambiguous/missing context (including a medication belonging to another person, a brand-vs-generic name, a misspelling, a planned medication, and multiple concurrent medications).
- **Reproducible pipeline** — consultation text → structured extraction → synonym normalization → negation & temporality classification → **deterministic** lookup in the evidence set → result state.

## Safety boundary (non-negotiable)

The language model may **only** extract: hormonal product, other medication, normalized names, status (current / historical / planned / negated / uncertain), explicitly stated dose or route, missing information, and whether a lookup should run.

The model **never** generates interactions, consequences, mechanisms, evidence levels, citations, severity, or any treatment/dosing advice. All visible medical content is loaded verbatim from `src/data/evidence_records.json`. The app never shows a green "safe" state and never claims that no interaction exists.

## Quick start

```bash
npm install
npm run dev          # http://localhost:5173  (Demo mode, no API key needed)
```

Other commands:

```bash
npm run benchmark    # runs the evaluation, writes src/data/benchmark_results.json
npm run test         # unit + integration tests (Vitest)
npm run build        # production build
npm run typecheck    # tsc --noEmit
```

## Modes

- **Demo mode** (default) — deterministic cached extraction for the five demo cases and a rule-based extractor for free text. Works with **no API key**.
- **Live mode** — calls a server-side/serverless endpoint (`/api/extract`) that performs a structured-output model call. Model and key are read **only** from environment variables (`OPENAI_API_KEY`, `OPENAI_MODEL`) and are never exposed in browser code. See `.env.example` and `api/extract.example.ts`.

## Data files

| File | Purpose |
| --- | --- |
| `src/data/evidence_records.json` | Six physician-reviewed interaction records (source of truth for all medical content). |
| `src/data/demo_cases.json` | Five scripted demo cases with cached deterministic extractions. |
| `src/data/benchmark_cases.json` | 20 synthetic cases with immutable gold labels. |
| `src/data/benchmark_results.json` | **Generated by the runner** — never hand-written. |

## Stack

React + TypeScript + Vite + Tailwind CSS, with hand-built shadcn-style UI primitives. Tests with Vitest; end-to-end checks and screenshots with Playwright.

## Limitations (read before drawing conclusions)

Small prototype dataset; synthetic benchmark; manually curated gold labels; no clinical validation; not comprehensive; no treatment recommendations. **Benchmark performance does not establish clinical safety. Absence of a record is not evidence of absence of an interaction.**
